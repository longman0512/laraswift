InstallationLast updated: 3 weeks ago

Installation instructions and set up for Laraswift.

01. Server Requirements

Below is a list of server requirement to install LaraSwift Application

  • PHP >= 7.2.0
  • BCMath PHP Extension
  • Ctype PHP Extension
  • JSON PHP Extension
  • Mbstring PHP Extension
  • OpenSSL PHP Extension
  • PDO PHP Extension
  • Tokenizer PHP Extension
  • XML PHP Extension

02. Install on Localhost


Laraswift Installation on Localhost

After downloading and extracting the ZIP archive file to your computer, create a database on your local server where all database tables will be created & stored. Next, ensure that permission level for bootstrap, storage & public folders is 777 and also permission for .env file should be 777 before proceeding to installation. Click here to see more about changing directory/file permission level.


﻿Quick Installation Steps (Use these steps for a quick installation of Laraswift on your local server)

  1. Please carefully follow all the steps listed below one after the other.
  2. Extract Laraswift zip archive file into a folder
  3. Make sure to create a database where all the database tables will be stored
  4. Make sure that permission level for bootstrap, storage & public folders is 777 and also permission for .env file should be 777.
  5. Open a terminal and navigate to the folder Laraswift project is extracted to and start local server.
  6. Run the command php artisan serve to start server.
  7. Open your browser to start installation. Go to  http://localhost:8000/ or the app url on your computer to access the installation wizard.
  8. When the installation wizard opens, click on Check Requirement to check for the required php version and php extensions.
  9. Click on Check Permission it checks for folder permissions, if all required folders for installation are with the 777 permission level click on Configure Environment. Cick here to see how to change directory/file permission level.
  10. Click on Configure Environment to configure all environment variables (.env) including the database details; ensure database details are accurate and  then click on Save .env. NOTE: DO NOT click on Save and Install yet.  
  11. Finally, click on Save and Install. If installation is successful, exit and you should be greeted with the Laraswift login screen. Your default admin credentials are: Username: admin, Password: Password1. Note that credentials are case sensitive.
  12. If there was an error during installation or there is need for re-installation, navigate to the storage folder and delete the installed file you find there. After that, open http://localhost:8000/ or the app url on your computer in your browser to begin the installation again, ensure to carefully follow all instructions.

03. Install on a Live Server


Laraswift Installation on Live Server

After downloading and uploading the ZIP archive file to your server, create a database on your server where all database tables will be created & stored. Next, ensure that permission level for bootstrap, storage & public folders is 777 and also permission for .env file should be 777 before proceeding to installation. Click here to see more about changing directory/file permission level.


﻿Quick Installation Steps (Use these steps for a quick installation of Laraswift on your live server)

  1. Please carefully follow all the steps listed below one after the other.
  2. Extract Laraswift zip archive file into your server directory, i.e where you want laraswift to be installed.
  3. Make sure to create a database where all the database tables will be stored
  4. Make sure that permission level for bootstrap, storage & public folders is 777 and also permission for .env file should be 777.
  5. Open your browser to start installation. Go to  https://yourdomain.com/ to access the installation wizard.
  6. When the installation wizard opens, click on Check Requirement to check for the required php version and php extensions.
  7. Click on Check Permission it checks for folder permissions, if all required folders for installation are with the 777 permission level click on Configure Environment. Cick here to see how to change directory/file permission level.
  8. Click on Configure Environment to configure all environment variables (.env) including the database details; ensure database details are accurate and  then click on Save .env. NOTE: DO NOT click on Save and Install yet.  
  9. Finally, click on Save and Install in the installation wizard to complete installation. If installation is successful, exit and you should be greeted with the Laraswift login screen. Your default admin credentials are: Username: admin, Password: Password1. Note that credentials are case sensitive.
  10. If there was an error during installation or there is need for re-installation, navigate to the storage folder and delete the installed file you find there. After that, open http://yourdomain.com/ in your browser to begin the installation again, ensure to carefully follow all instructions.

UsageLast updated: 2 months ago

This is a step by step instruction on the app usage

Activity log

Admin Activity-Log

The admin activity log registers all activities carried out.


User Activity Log

The user activity log registers all the activities carried out by a particular user.




Application Settings

﻿﻿Settings

The application settings is the control room for all functionality of the application. Controlled functionalities include App name, App logo, App them, Payment settings, Authentication settings and Application backup (Application file and Database)
App Name
Application name can be updated by admin


App Logo

Application logo can be updated by admin


App Theme

Application top-navbar can be update by admin


Payment Settings

To activate payment admin is required to update Stripe Publish Key and Stripe Secret Key to enable the payment in the platform.


Authentication Settings

Authentication settings the admin can enable/disable Two Factor Authentication, Captcha, and Email Verification.


App Backup

App backup the application files and database can be backup and downloaded.




Authentication and Access

﻿Login Page

After all proper installation users will be redirected to login. Login has multiple methods of authentication. Users can login using username/email and password also Social Authentication( Facebook, Twitter, Google ) with re-Captcha robot login protection (if enabled by admin).


Registration Page

Registration form for users to create an account can be accessed from the login page by clicking Register a new membership. Registration form can be protected using re-Captcha (if enabled by admin).

Note: If email verification is activated by admin in the application settings then after registration email will be sent to the new members for verification.Also applies to new members who sign-in with Social Authentication.

Password Reset

Password reset for forgotten password. Can be accessed from the login page by clicking I forgot my password. 

After entering and click on send password reset link email will be sent to user

Two Factor Authentication

If two factor authentication (2FA) is enabled by admin in the Application Settings, then every user can activate 2FA in their profile. More details of personal 2FA activation will be discussed in the Profile Section. 


Verification code is obtained from Google Authenticator app, every user is required to install the app to activate 2FA which automatically generates codes for users whenever they need to sign-in.
Note: Google Authenticator is available on Andriod and iOS



Contents

Premium Content

Premium content can be controlled by enabling/disabling of payment, if payment is activated only subscribed users will be able to access premium contents while if payment is disabled every user has access to premium content.


Dashboard

Admin Dashboard

The admin dashboard is more like the control room of the application that houses every functionality of the application. On the admin dashboard there are some of the application statistics which include total users, roles, permission, activity log, chart of users registration history and recent payment transactions (if payment is enabled by admin). This dashboard is strictly for the default administrator and any other user with the admin role



Default Dashboard

﻿The default dashboard is to give users a quick link to every section of the system like the activity-log, profile, article content published by admin and subscription(if payment is enabled by admin). Other privileges are granted to the user depending on the permissions assigned to the user role. 





Payment Management

Payment

Lara-swift payment is built to manage stripe recurring payments, managing subscription plans,subscribed users andincome balance if payment is enabled by admin  in the application settings.

Subscription Plan Management

Subscription plan can be created, edited and deleted and has its concurrent effect in the Stripe Dashboard.


Creating Plan

Creating a plan, the following are the required plan name, plan description, amount, plan interval (daily, weekly, monthly and yearly), for the plan interval count it specifies the rate of the interval that is it specify the period before the next charge. Assuming interval is selected is monthly and interval count is 2, therefore the subscription plan will be charging users every 2 Months before the next charge. Also a plan trial period is allowed for each plan in which a user tries the Premium Content.

Editing Subscription Plan: Only the name and description of a subscription plan is allowed to be edited.


Payment Income Management

List of all payments made by users on the platform.


Subscribed User Management

Subscribed user management gives a list of subscribed users indicating subscription detail of user also admin can cancel user subscription and also generate payment invoice of a subscriber.





Role and Permission Management

Roles management

Every user is assigned a particular role and different permissions are attached to the role which determines the privilege of the user to access different parts of the application. By default Lara-Swift has admin and user roles.


To manage permissions attached to a particular role add permission button is clicked

Permission management

Permissions are attached to roles and they determine the privilege access of each role. Lara-swift has some default permissions user-activity, manage-user, manage-permission, manage-role, manage-setting, manage-payment-gateway.


NOTE: All default permissions cannot be deleted.



User Management

Admin User Management

The admin role has a permission for users management manage-users. The admin has the ability to impersonate any user and also update users account and login details.


For admin to impersonate any user the impersonate button is clicked for that particular user as seen below.


To exit user impersonation Exit Impersonation is clicked on the top-right of the nav-bar.




User Profile

Profile

User profile is where logged in users can view and update their profile details such as the Bio-data, Login details and Two Factor Authentication (2FA)  if enabled by admin.

If 2FA is disabled the user profile has 2 tabs to update profile, Account detail and Login details


If 2FA is enabled by admin the user profile contains 3 tabs which include the addition of Two Factor Auth tab to activate 2FA in the logged in users profile.

Uploading Profile Avatar


Activation and Deactivation of Two Factor Authentication

Activating the 2FA Step by Step instruction if gotten by clicking Setup Instruction


Click on Activate 2FA to activate, a QRCode will be generated and scan the QRCode using Google Authenticator or Authy mobile app.


Enter the code generated code and click on Enable 2FA to enable 2FA. To disable it, generated code from Google Authenticator or Authy which you used in Enabling it is required and user account password.

Change Directory/ File Permission

﻿﻿For the installation wizard to run properly you need to give 777 level permission to bootstrap, storage, public. Before installation ensure .env file is given 777 permission and after installation is completed change the permission level of .env file back to 644.  Directory/File permission can be changed using the following command-line in CLI 

Example:

﻿chmod -R 777 public

﻿chmod -R 777 storage

﻿chmod -R 777 bootstrap

﻿chmod -R 777 .env

After changing the file permission level the steps below to run installation wizard.
